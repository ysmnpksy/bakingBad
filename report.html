Report by Yasmin Paksoy

Introduction

This site is a recipe website inspired by the tv show Breaking Bad. All of the recipes and drinks are ones that are were either made on the show, such as Walt's Birthday Platter, or are inspired by the scenes on the show, such as Walt's Burried Barrel Cake.

Background

The target audience for this site is fans of the show Breaking Bad. This includes people of any ages.

There arent many stakeholders of this website as it is a blog. The few stakeholders include the owner of the blog, myself, the creators of the recipes that are featured on the website and Breaking Bad fans who use the website.

The idea for this website came from the book Game of Scones - All Men Must Dine. This book is a recipe book inspired by the TV shows Game of Thrones. Its aimed at Game of Thrones fans and is full of recipes of the food mentioned in the show and desserts inspired by some of the most iconic scenes. The book consists of images of the desserts with the ingredients and directions listen beside. All the text in the book contains witty jokes and puns about the show to engage fans. This was the main inspiration for my website; I wanted to make a site where fans of one of my favourite shows, Breaking Bad, could find recipes that remind them of some of the most popular scenes from the show.

The layout for the recipes and drinks page was inspired by the website Baking Mad. This is a baking website which consists of several different pages of different categories; each page has rows of square images with the name of the dessert underneath, which takes you to the recipe when clicked. This was the main inspiration for the way i decided to lay out my website, as it has a very clean look and is not confusing for the user.

Another source of inspiration for this website was the official Breaking Bad website. I used this as inspiration for the colour scheme, logos and font style. The main colour on this page and on the Breaking Bad logo is a dark green, and the logo is inspired by the periodic table. I used this as inspiration and duplicated this into my website.

Planning

To ensure accessibility on my website I did many different things, the first of which was to use a dyslexia-friendly font. The font I chose was Arial, which is a sans-serif font, making it easier for dyslexic people to read. I also made sure to include alt tags on all of my images, so that visually impaired users would still be able to understand what the image shows. I made sure not to use colours which were too similar and difficult to read also; all of my text is black, with headings being a dark green which contrasts well with the white background.

To ensure usability in my website, I made sure to be consistent; all of the pages have the same header and footer and use the same font styles and colours so users get to have a consistent experience and don't have to waste time trying to figure out new layouts and elements.

Additionally, to ensure usability I did my best to make my site responsive; most buttons have a hover effect so users are able to tell that they are clickable. For example, the buttons on my recipes page have a shadow when hovered over and the buttons that control the slideshow on the home page are animated and slowly turn a darker grey when hovered over.

Another thing I did to ensure usability was to use @media tags to make sure some aspects, such as the navigation bar and margins, change when the website is viewed on a small browser size. This ensures that the website is still usable if users want to have two windows up on their browser or are viewing the website on a smaller tablet screen.

Implementation

Modals

When coding this website, I wanted to use modals for the recipes and drinks pages. To do this, I researched and found this code from w3schools, which gave me a basic template to work with. I lifted the CSS straight into my code and left most of the colours as they were neutral tones which worked well with my colour scheme. Using the example on the website I wrote the HTML and copied over the Javascript into my page. However, as I tried to put multiple modals into my page, this code didn't work. I then had to research how to make modals work when there was multiple on a page and found this post on StackOverflow. I used the JavaScript on this post and adapted it to my HTML, adding multiple different lines of code to make sure my modals opened and the close buttons worked.

I then added CSS to edit my buttons, so that the buttons that made up the modals fit in well with my website. I used basic code from this w3schools article. I changed the colours and adding more code to change the width to match the images on my recipes page to give it a cleaner look.

Collapsible Box

I wanted to use collapsible boxes on my tips page so that the page looked cleaner and wasn't only full of text. To do this, i used this code on w3schools. I copied over the CSS but edited it, changing attributes such as padding, margins and width. I left most of the colours the same as they were neutral tones that looked well with my colour scheme. I then used the example HTML on the site to write my own, lining mine up in two columns. Lastly, I copied over the JavaScript from this article to my page, to make my buttons work.

Slideshow

I wanted to include a slideshow on my home page so that users could have a look at the multiple different recipes of baked good the websites includes in a way that was more interactive and pleasing o the eye than having a gallery of images. I used this page w3schools to do this; I based my CSS on the CSS example on the tutorial, changing most things such as the width, padding and colour of the previous and next buttons, the colour the buttons turned when hovered on, and the positioning, size and colour of the caption text boxes, creating three different ones so I could have the text on a different position on each image. I also added dots to the bottom of my slideshow as the example had done to allow users to see the which image they were on and how many there are in total. I then added my own HTML, basing it on the tutorial code and copies over the JavaScript from the example to make my slideshow work.

Resources used

The main library I used when needing references or examples was w3schools. I also used stackoverflow to search for answers on problems i was having. While writing my code, I used the validator to make sure I didnt have mistakes in my code and fixed the one I had made.

Evaluation

What worked well

I think the modal boxes was the best part of my website; using modal boxes instead of linking each button to a new page which had the recipe details would have left the website very messy looking and complicated to use; users would have had to go back or navigate back to the recipes page using the navigation bar. An alternative to this would have been listing the images and the directions side by side, however, this would leave the page with too much text to be aesthetically pleasing. The modal boxes were the perfect compromise; there didn't have to be too many pages created but the recipes page still looked minimalist.

Another thing that worked well in my website was the slideshow; giving users interactive images with links to other pages is a good way to introduce them to a website; it allows users to get a sense of what the website is about while letting them explore further and without too much thought. The slideshow also looks aesthetically pleasing and having multiple images in such a small amount of space allows the page not to look crowded.

Another thing that works well on my website is how it adapts to resizing; I've used percentage values for the width of most elements and vw units for the font size of most text. These allow my website to adapt nicely when the screen size is adjusted. I've also included a scend navigation bar which consists of a drop down menu. This replaces the main one when the screen size gets too small for it to fit, which I've done using @media tags.

Improvements

Unfortunately, the footer on my Baking Tips page didn't work. As the page size grew when the collapsible was opened, the footer started to go over the text. To combat this, I tried to adjust the height of the page so it grew automatically but was unable to do so. I also tried including a sticky footer on this page so that it was always at the bottom, no matter what the height of the page was, but the footer started appearing at the top. If I had more time I would try to make the footer work on this page so that it was at the very bottom of the page and the page height grew as the collapsibles were opened.

If I had more time I would also try to change the essentials page; I think it would be more pleasing if it was more interactive. The page, while getting the relevant information across, is very basic in its layout. It may become difficult to navigate as more items are added to the page. I would have liked to have categories which open new pages with the essential products listed as buttons opening collapsibles or modals.

Another thing I think can be improved on this website is the overall decoration; while I've tried to keep it clean and sophisticated, I think it may look too simple. A little more colour or boarders around containers may have made it look more luxurious. I also think that the website doesn't represent Breaking Bad as much as I would have liked; the show has a much darker atmosphere, which is something I was cautious of including in the fear it would make the website look too dark and make it difficult for users to look at and read the text.
